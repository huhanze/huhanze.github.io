类的自动转换和强制类型转换

2013-05-27
  • C++

在C++中，将一个标准类型变量的值赋给另一种标准类型的变量时，如果这两种类型兼容，则C++自动将这个值转换为接收变量的类型。

如：

long number = 10;           // 将int数值10转换为long类型

double value = 5;            // 将int类型转换为double类型

int value = 8.88;            // 将double类型转换为int类型，但该转换损失了0.88精度，转换结果为8

对于这些基本类型相互兼容，C++编译器能自动转换，但是C++不能自动转换不兼容的类型，如，下面的转换就是非法的：

int *p = 8;

左边是 int * 类型的变量，右边是一个数值，虽然计算机内部可能只用整数值来表示地址的值，但从概念上说，指针和数值是完全不同的两类概念；虽然C++不能自动转换，但有一个方法可以做到，那就是强制转换。

int p = (int )8;

该语句将8强制转换为 int *,将p的值设为8，虽然该做法可能没有任何意义，但通过强制转换也能达到目的。

第一句 Point point1；创建一个Point对象，并初始化x = 0.0, y = 0.0;

那么可以point1 = 5.0; 这么做吗？答案是肯定的，这句语句能够能过编译，并且执行完这句后point1 - x:5.0, y:0.0;

为什么呢？这其实就是C++将5.0隐式转换为了Point类型，在Point类声明中，其中声明了一个Point(double x);的构造函数，那么在执行point1 = 5.0时，编译器隐式地将该语句当做point1 = Point(5.0)来处理的，这一操作是C++自动进行的，不需要显示转换。

C++将类的构造函数用作自动类型转换是一个不错的特性，然而在大多时候这个特性却令人头疼，很多时候不希望被自动转换。因此，C++有一个关键字 “explicit” 关闭了自动转换的特性，也就是说在声明构造函数时在函数前面加上explicit关键字即可，如：

那么，添加explicit关键字后，point1 = 5.0；编译器不允许通过了，这样就必须显示的转换，point1 = Point(5.0);或者point1 = (Point)5.0;

注意：函数原型化提供的参数匹配过程，允许使用Point(double)构造函数来转换其他数值类型，如：Point(10);是将int类型10先转换为double类型10.0，然后将double类型10.0通过构造函数Point(double);转换为Point类型。

然而，当且仅当转换不出现二义性时才会进行这样的转换。也就是说如果Point类还定义了构造函数Point(long),编译器将拒绝执行Point(10),因为int类型可以被转为long或者double两种类行，因此，在这种情况下会出现二义性。

注意 ：

  1. 转换函数必须是类方法。

  2. 转换函数不能指定返回类型。

  3. 转换函数不能有参数。

  4. 当定义了两种或更多的转换时，仍可以显示用强制类型转换来指出要用哪一个转换函数。

  5. 最好使用显示转换，谨慎地使用隐式转换函数。
