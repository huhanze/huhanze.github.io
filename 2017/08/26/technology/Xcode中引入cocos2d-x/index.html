Xcode中引入cocos2d-x

2017-08-26
  • cocos2d

1. 打开Xcode新建一个Demo项目

2. 去cocos2d-x官网下载SDK

去cocos官网下载cocos2d-x

新建一个cocos2d文件夹，将下载的SDK中的这些文件拷贝一份到新建的文件夹中：

将选中的文件拷贝到cocos2d文件夹

3. 导入cocos2d

将新整理的cocos2d中的 build -> cocos2d_libs.xcodeproj 拖入到Demo的目录结构下

此时需耐心等上一分钟，因为拷贝cocos2d文件到Demo目录需要一点时间
，或者先将cocos2d提前拷贝到工程下，在导入到Demo工程目录。

4. 添加相应的配置项

选择工程的Target：

选择 Build Setting 选项, 添加相应的文件引入信息

同样选择 Project 引入相同的 Header Search Paths

然后导入需要的依赖库：

接下来设置编译环境：

此时，相关工作已经准备的差不多了，但还有一个地方需要修改，在 build Setting 搜索栏搜索 bitcode 并设置为 NO

然后编译一下，如果不出问题，编译会通过。

5. 导入资源

在项目的工程目录创建一个存放资源的文件夹， 我这里命名为 “Resources” ,然后将所有的图片资源放到该文件夹下，并将该文件夹拖入Xcode中即可

此时cocos2d导入已完成，可以开始测试啦。

###注意 ：

iOS工程下有AppDelegate文件，所以在定义cocos2d的Application代理时不能和当前工程下AppDelegate重名，改哪一个都行，看个人喜好，总之不能重名。

iOS下将OC的 .m文件后缀改为 .mm就可以OC、C++混编啦。
