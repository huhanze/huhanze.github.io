OC对象本质

2017-12-18
  • iOS底层原理

前言

在iOS开发中，我们常使用Objective-C这种面向对象语言来进行开发，而代码真正执行的时候，其实经过了几层转换，OC代码会转化为runtime下的c/c++代码，进而转为汇编，最终转化为机器语言来执行。其实说白了，Objective-C的对象其实就是指向结构体的指针，OC对象的方法在调用时会转化为相应的函数来执行。也就是说，完全可以用C和C++进行开发，但为什么苹果要推出OC呢？其实就是方便广大开发者，苹果写好底层代码，然后提供简单的接口和规则，开发者开发的难度大大降低，苹果在这方面给个大大的👍！废话不多说，下面直奔主题。

OC对象创建会调用class_createInstance()函数，而在该函数中调用了 _class_createInstanceFromZone()函数。接下来看看 _class_createInstanceFromZone()函数的实现：

这段实现中分配内存的代码其实就是 obj = (id)malloc_zone_calloc ((malloc_zone_t *)zone, 1, size); 和 obj = (id)calloc(1, size); , 传入的size就是size_t size = cls->instanceSize(extraBytes);,也就是说 instanceSize(extraBytes)就是实际分配的大小，接下来就是看看 instanceSize() 函数的实现。

在 instanceSize()中，size 就是对齐后的实例变量的大小与额外空间(这儿实际上是0)大小，当size的值小于16时，size将被置为16，所以一个NSObject对象的占用16个字节的空间，其中isa占用8个字节。

NSObject 调用 + (id)alloc 方法分配内存大小流程：

[NSObject alloc] -> _objc_rootAlloc(self) -> callAlloc(cls, false/*checkNil*/, true/*allocWithZone*/) -> class_createInstance(cls, 0) -> _class_createInstanceFromZone(cls, 0, nil) -> cls->instanceSize(extraBytes)获取要分配的内存大小 -> 分配内存。

4. OC对象的分类

OC对象主要分为3种：实例(instance)对象、类(class)对象 和 元类(meta-class)对象

实例对象：我们通过类调用 + (id)alloc 方法生成的对象就是实例对象，如

dog1 和 dog2 就是两个不同的实例对象，被分配两块不同的内存单元。它们都存储了 isa 、_age 、_weight 和 _legs 成员变量，各是各的，互不影响。

类对象：通过调用 + (Class)class 方法、或者 object_getClass() 函数，返回的对象，如

执行结果: class1 : 0x1000021b0 class2 : 0x1000021b0 class3 : 0x1000021b0 class4 : 0x1000021b0 class5 : 0x1000021b0

执行的结果，class1 ~ class5都指向同一个地址，说明了每个类对象只存在一个。
类对象在内存中存储的主要信息包括 isa指针、superclass指针、属性信息、对象方法列表、协议列表信息、成员变量列表信息等。

元类(meta-class)对象：什么是元类对象呢？其实就是类对象的isa指针所指向的对象，元类对象中存储了类方法列表信息，当然元类对象也有isa指针和superclass指针等信息，在内存结构上和类对象是一样的，只是分工不同。每个类的元类对象也只有一个。

执行结果：metaClass1 : 0x100002188 metaClass2 : 0x100002188

5. isa指针和superclass指针

isa指针

isa 指针在OC对象中有着至高无上的地位，对象的相关操作依赖于它。实例对象在调用方法时，先通过其isa指针找到类对象，然后去类对象的方法列表中去调用相应的方法。类对象通过其isa指针找到元类对象进而调用类方法。

实例对象、类对象、元类对象都存在isa指针。

superclass指针

superclass指针只存在于 类对象 和 元类对象中， 当子类调用父类的方法时，通过 superclass指针找到父类从而调用对应的方法。

小结：

  1. 实例对象 的 isa 指向 类对象。

  2. 类对象 的 isa 指向 元类对象。

  3. 元类对象 的 isa 指向基类的 元类对象。

  4. 类对象 的 superclass 指向父类的 类对象，基类 类对象 的 superclass 指针为nil。

  1. 元类对象 的 superclass 指向父类的 元类对象，基类 元类对象 的 superclass 指向基类的 类对象。
