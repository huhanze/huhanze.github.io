2018-12-28

1. 简单介绍

NumPy是用科学计算的Python第三方库，主要包含了这些功能：

  • 强大地处理多维数组数据对象。
  • 有效处理复杂的数据。
  • 可用于集成C/C++代码。
  • 用于线性代数、傅里叶变换、生成随机数等。

除此之外，NumPy可作为一般数据的有效多维容器，任何数据类型都可以被定义。这就允许NumPy无缝快速地整合各种数据库。

NumPy的主要对象是多维数组，数组中有一些列相同类型的元素，并且可以通过索引(正整数)获取元素。在NumPy中，维度又被称为轴。如：

2.2 zeros()、ones()、empty()函数

通常，定义一个数组时，并不确定数组的元素，但知道要定义的数组大小。因此，NumPy提供了几个函数来初始化占位符。这些函数减小数组的扩展、昂贵的开销。 zeros() 创建一个数组并将元素都初始化为0，ones() 创建一个元素值都初始化为1的数组，empty() 函数则初始化的数组的元素值由内存的状态随机生成，默认情况下，这些函数的数据类型为 float64 。

由于浮点数的精确度是有限的，当arange使用浮点数做参数时，通常获得的数值元素不能预料。对于这个问题，通常使用 linspace(start,stop,num=50,endpoint=True,retstep=False,dtype=None) 函数作为想要接收的数值元素为参数。

2.3 数组的打印

在NumPy中，数组的打印结果是以列表嵌套的方式显示，布局有以下特点：

  • 最后一个轴的打印是从左至右
  • 第二维至最后一维之间的打印是从上之下
  • 剩余的部分也是从上之下打印出来，每个部分使用空行隔开

也就是说一维的数组按行打印，二维数组按矩阵打印，三维数组以矩阵列表的形式打印。

如果需要被打印数组的元素数量过多，NumPy会自动忽略数组的中间部分，只显示开头和结尾的元素。

如果要禁用这个行为，强制NumPy打印完整的数组可以使用 set_printoptions() 改变打印结果。

3. 基本操作

3.1 NumPy中数组的算术运算

与很多矩阵相关操作语言不同，Python3.5版本以后，矩阵积的运算是使用 @ 操作符或 dot() 函数，而并不是和其他语言一样使用 * 运算符。

有些操作，如 += 和 *= , 则是修改一个已经存在的数组，而不是创建一个新的数组。

当对两个不同的数组类型操作时，数组结果类型通常以更精确的一方类型。

很多一元运算，如数组内元素求和运算，是作为 ndarray 的方法实现的。

默认情况下，这些操作是应用到数组的一列数值，无视数组的结构。然而可以通过 axis 参数来指定数组中哪个轴的数组操作。

3.2 索引、切片、迭代

先来看看以为数组的操作

多维数组可以在每个轴使用index操作，并且通过逗号分隔元组显示。

NumPy中允许使用如 b[i,...] 来表示一个拥有很多列的数组，例如：x 是一个拥有5个轴的数组。

  • x[1,2,...] 等价于 x[1,2,:,:,:]
  • x[...,3] 等价于 x[:,:,:,:,3]
  • x[4,...,5,:] 等价于 x[4,:,:,5,:]

3.2 shape相关操作

数组的shape可以通过一些命令来改变，如下的几个命令返回修改后的数组，但并没有改变原数组：

a.ravel() 返回了一个一维数组，a.shape() 返回shape改变后的数组，a.T 返回原数组的转置数组。

ravel()返回的数组元素的顺序一般按照C-style排列的，也就是说最右边的元素先变化，那么a[0,0]后边的元素就是a[0,1]。如果数组的shape被改变成其它的shape，该数组仍被视为C-style。正常来说，NumPy按照此顺序储存被创建的数组，ravel()不需要拷贝它的参数，如果该数组被其他的数组切分或以不寻常的方式创建，可能会需要拷贝相应的参数。

3.3 不同数组间的叠加

一些数组可以沿不同的轴进行叠加：

column_stack函数按列将数组叠加成一个二维数组。对于二维数组等价于hstack。

另一方面，row_stack函数等价于vstack。一般来说，超过2维的数组，hstack沿第二轴，vstack沿第一轴,并且concatenate允许可选参数指定轴。

3.4 数组的分割

hsplit函数可以将一个数组沿水平轴进行分割，或者指定一个数值做等值分割。

vsplit函数将数组垂直分割，array_split函数可以指定相应的轴分割数组。

4. 拷贝和视图

在对数组操作时，有时候需要拷贝一个新的数组，而有时不需要拷贝，有以下三种情况。

4.1 不需要拷贝

简单的赋值操作不会拷贝数组对象。

Python对可变对象按引用传递，那么函数的调用不会做拷贝操作，上述的b和a指向同一个内存地址。

4.2 View 或浅拷贝

不同的数组对象可以共享相同的数据，view方法创建一个新的数组对象，但共用原数组的数据。

4.3 深拷贝

copy方法完全拷贝数组及其数据。
